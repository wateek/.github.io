Problem #1: COIN CHANGE

Background Information:

The Acme Software Company wants to develop a program that will determine the minimum number of coins needed for any value between 0 and 99 cents. They want the program to be general enough to allow for the input of up to 10 types of coins. For example, in the U.S., we usually consider coins of 1, 5, 10, 25 and 50 cents. The minimum number of coins needed for 71 cents is 4: a 50, 10, 10, and 1. In Bionia, there are 1, 2, 4, 8, 16, 32, and 64 cent coins. The minimum number of coins needed for 71 cents is 4: a 64, 4, 2, and 1.



Programming Problem:

There will be one integer per input line. Also, CN will always be equal to 1 so there will always be a solution.
